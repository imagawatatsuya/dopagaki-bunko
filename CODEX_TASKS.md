# CODEX_TASKS.md

現在の `dopagaki-bunko` を前提に、Codexへ順番に渡すための作業キューです。
MVPの作り直し手順ではありません。いまある静的アプリを保守・拡張する前提で使ってください。

一度に全部やらせず、上から1つずつ実行させてください。

## 0. 最初に渡す共通指示

```text
AGENTS.md、SPEC.md、docs/dependencies.md、docs/ui-pages.md、CODEX_TASKS.mdを読んだうえで作業してください。
このリポジトリは、GitHub Pages の main / root 公開を前提にした既存の静的Webアプリです。ゼロから作り直さず、現行実装の延長として変更してください。
外部依存、npm install、外部CDNは禁止です。
```

## 現在の前提

- 画面は `ホーム / 本棚 / 追加 / 設定` の4画面が実装済み
- ホームは出来事ベースのタイムラインで、`作品追加` `最新しおり` `ふせん追加` を混在表示する
- 本棚は `読書中 / 未読 / 読了` タブと、`しおり一覧 / ふせん一覧` 導線を持つ
- 作品ページは `右下断片番号` の `なし → しおり → ふせん → なし` 循環トグルと、`原文終端 / 読了` 切替を持つ
- 断片個別ページの操作は `ふせん / しおり / ふせんメモ`
- 既定ではサンプル作品を自動投入しない
- 保存ストアは `works, fragments, likes, bookmarks, readingStates, settings`
- 青空文庫検索は同梱 `data/aozora-catalog.json` を使い、作品ZIPは手動で取得してZIP取り込みする
- 公開前検証は `verify-self-tests.ps1` と `verify-pages.ps1` を通す

