# 青空文庫取り込みフロー

## 目的

青空文庫のルビありZIPを直接取り込み、作品アカウントと本文断片を生成する。

## 処理順

1. 同梱の圧縮作品一覧読込または手動ZIP選択
2. ZIPサイズ検査
3. ZIP内ファイル一覧取得
4. `.txt` 抽出
5. store/deflate判定
6. ファイル展開
7. txtならShift_JISデコード
8. ヘッダ・フッタ除去
9. 外字注記の一部解決
10. 見出し注記の構造化
11. ルビ変換
12. 圏点変換
13. HTML安全化
14. 断片化
15. プレビュー
16. 保存

現状の外字注記は、青空文庫の `※［＃「...」、第3/4水準...］` や
`※［＃二の字点、1-2-22］` のように面区点番号を持つものを、JIS X 0213 の
面区点対応表で実字へ置換する。

`※［＃「...」、U+....、...］` のように Unicode 番号が明示されたものも
実字へ置換する。

面区点や Unicode 番号を解決できない注記は、原文のまま残す。

見出し注記は、大見出し・中見出し・小見出しを表示へ反映しつつ、
後で目次へ使える `outline` 情報として作品データへ保持する。

`［＃「序」は中見出し］` のような注記後置型に加え、
`［＃中見出し］登場人物紹介［＃中見出し終わり］` のような開始/終了タグ型も見出しとして扱う。

字下げ注記は見出しの補助情報としてだけ使うのではなく、
`［＃３字下げ］...` や `［＃地付き］...`、`［＃地から１字上げ］...` のような
行頭レイアウト注記については、線形読書でも意味が分かる軽い段下げ・右寄せへ変換する。

ただし `15字下げ` のような値を、そのまま大きな余白として厳密再現はしない。

`［＃改ページ］` や `［＃改丁］` のような版面区切り注記は、
線形読書ではページ境界そのものを再現できないため、区切りラベルとして表示する。

未対応の注記-only行も、初期実装では生の `［＃...］` をそのまま見せず、
短い区切りラベルへ落として読書阻害を減らす。

本文中の `「...」に横組み` や `「...」に割り注`、および対応する範囲注記は、
初期実装ではインライン span として軽く表示変換する。

## 本文範囲推定

青空文庫テキストには冒頭の注記説明と末尾の底本情報が含まれる。
自動判定は完全ではないため、保存前にプレビューで確認できるようにする。

## 検索取り込み

青空文庫の作品一覧は、同梱の圧縮スナップショットを読み込んで `aozoraCatalog` ストアへキャッシュする。

- 検索対象は作品名、作品名読み、著者名、著者名読み
- 検索結果カード全体はリンク化せず、公開条件を満たす `https://www.aozora.gr.jp/cards/<数字>/files/*.zip` だけを `青空文庫ZIPを開く` として直接開ける
- ZIP を開けない作品や著作権注意作品では `図書カードを見る` を副リンクとして表示し、公開条件を確認してもらう
- `一覧を再読込` は同梱カタログを読み直すだけで、青空文庫サイトへの直接通信はしない
- 作品ZIPの取得は自動 fetch せず、ユーザーが保存した ZIP を既存の `作品を取り込む` 導線から手動で渡す
- 著作権フラグが `なし` 以外なら短い警告を表示する

## 失敗時表示

専門用語を避け、ユーザーが次に何をすればよいかわかる文にする。
