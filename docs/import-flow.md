# 青空文庫取り込みフロー

## 目的

青空文庫のルビありZIPを直接取り込み、作品アカウントと本文断片を生成する。

## 処理順

1. 同梱の作品一覧JSON読込または手動ZIP選択
2. ZIPサイズ検査
3. ZIP内ファイル一覧取得
4. `.txt` 抽出
5. store/deflate判定
6. ファイル展開
7. txtならShift_JISデコード
8. ヘッダ・フッタ除去
9. 外字注記の一部解決
10. 見出し注記の構造化
11. ルビ変換
12. 圏点変換
13. HTML安全化
14. 断片化
15. プレビュー
16. 保存

現状の外字注記は、青空文庫の `※［＃「...」、第3/4水準...］` や
`※［＃二の字点、1-2-22］` のように面区点番号を持つものを、JIS X 0213 の
面区点対応表で実字へ置換する。

`※［＃「...」、U+....、...］` のように Unicode 番号が明示されたものも
実字へ置換する。

面区点や Unicode 番号を解決できない注記は、原文のまま残す。

見出し注記は、大見出し・中見出し・小見出しを表示へ反映しつつ、
後で目次へ使える `outline` 情報として作品データへ保持する。

字下げ注記は見出しの補助情報としてだけ使い、`15字下げ` のような値を
そのまま大きな余白として再現しない。

## 本文範囲推定

青空文庫テキストには冒頭の注記説明と末尾の底本情報が含まれる。
自動判定は完全ではないため、保存前にプレビューで確認できるようにする。

## 検索取り込み

青空文庫の作品一覧は、同梱のJSONスナップショットを読み込んで `aozoraCatalog` ストアへキャッシュする。

- 検索対象は作品名、作品名読み、著者名、著者名読み
- 検索結果の項目全体を図書カードリンクとして扱う
- `一覧を再読込` は同梱JSONを読み直すだけで、青空文庫サイトへの直接通信はしない
- 作品ZIPの取得はユーザーが図書カード側で行い、アプリには手動ZIP取り込みで渡す
- 著作権フラグが `なし` 以外なら短い警告を表示する

## 失敗時表示

専門用語を避け、ユーザーが次に何をすればよいかわかる文にする。
