# UIページ仕様

## ホームTL

目的：x.comの代替として縦スクロールで読む。

表示：

- 作品名
- 本文断片

表示しない：

- アイコン
- 投稿日時
- 操作アイコン
- エンゲージメント数

補足：

- 初回起動時に見本作品は自動投入しない
- ホームTLの `作品追加` カードを開くと、作品ページの先頭を表示する
- ホームTLの `最新しおり` と `ふせん追加` のカードを開くと、作品ページ内の該当断片へ直接ジャンプする
- カード上段は作品名に加えて原文2行目までを一行で示し、シリーズ作を見分けやすくする
- 読了の作品は低コントラスト寄りの静かな差分で区別する

## 断片個別ページ

目的：断片をじっくり読み、操作する。

表示：

- 作品名
- 著者名
- 本文断片
- 本文直下のメモ表示兼編集面
- 本文直下の `TLのこの位置へ戻る`
- 前へ
- 次へ
- ふせん
- しおり

補足：

- しおりは同一作品内で常に1つだけ
- 新しい断片でしおりを付けると、古いしおりは置き換わる
- ふせんは付箋として複数残せる
- ふせんには短いメモを添えられる
- メモは本文直下の表示面を押して、その場で追加・編集する

## 作品ページ

目的：作品全体の進捗と断片一覧、および現在のしおり位置を見る。

表示：

- 作品名
- 著者名
- 読了進捗
- 現在位置の追従ヘッダ
- 断片一覧
- しおり位置を開く
- 保存済みoutlineから開く簡易目次
- 本文サイズのその場調整
- 右下断片番号による `なし → しおり → ふせん → なし` の循環トグル

補足：

- `しおり` は輪郭主体で示す
- `ふせん` は背景だけ淡いハート調で示す
- `しおり位置を開く` は単体ページではなく、作品ページ内の該当断片へ直接ジャンプする
- 簡易目次は既定で折りたたみ、開くと保存済み見出し構造を静かに並べ、各項目から作品ページ内の該当断片へ直接ジャンプする
- 目次などから後半へ直接ジャンプするときは、先頭から全断片を描画せず、対象周辺だけを開く。必要なら `前の断片を読む` で手前へ広げる
- 簡易目次の末尾には `終端` を置き、作品ページの最下部にある `原文終端 / 読了` 位置へ直接移動できるようにする
- 作品ページ上部から、その作品の `ふせん` 一覧だけを開ける
- 本文サイズのその場調整ユニットは作品情報の右側に静かに置く
- 全断片表示後の末尾に `原文終端` を置く
- `原文終端` は押すたびに `原文終端 ⇄ 読了` を切り替え、色でも区別する

## ライブラリ

目的：取り込んだ作品、作品ごとの最新しおり、ふせんを管理する。

表示：

- `読書中 / 未読 / 読了` タブ
- 状態別の作品一覧
- 保存一覧への導線

補足：

- 各タブの作品カードは `…` メニューから削除できる
- 読書中タブの `…` メニューには `未読に戻す` を置き、読書状態としおりだけを外して未読へ戻せるようにする
- スマホでもPCでも同じ導線を使い、作品名の可読性を崩さない
- 作品カードでは、しおりに加えて `ふせん` 枚数も静かに見える
- 削除は作品本体と関連する断片・しおり・ふせん・読書状態をまとめて消す

## 検索

目的：青空文庫作品を探す、取り込み済み作品を探す、または公開 TXT URL、貼り付け TXT、手元ファイル/同一Wi-Fi上のPCから作品を取り込む。

初期ではローカル検索のみでよい。

補足：

- GitHub Pages 版の主導線は `TXT 公開URL` と `TXT を貼り付ける`
- `TXT を貼り付ける` の本文は未保存で閉じた手入力だけ一時保持し、保存済みの貼り付け本文や PC/URL 経由の受け取り本文は次回の新規入力欄へ戻さない
- 青空文庫検索結果では、公開条件を満たす公式 ZIP があれば `青空文庫ZIPを開く` を直接表示し、図書カードは副リンク `図書カードを見る` に留める
- ZIP保存後の取り込みは、結果カードごとにボタンを増やさず、画面下部の既存 `作品を取り込む` を使う
- `PCのURL` では `PCからプレビューを開く` を使い、PC上の中継ページ経由で本文プレビューへ戻す
