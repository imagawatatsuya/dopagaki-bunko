# SPEC.md

## 1. 目的

本アプリは、x.com依存を弱めるための代替閲覧アプリである。
x.comのように短い投稿を次々に眺める行動を、青空文庫作品の短い断片を読む行動へ置き換える。

これは高機能な青空文庫ビューアではない。
縦スクロール習慣を文学断片読書へ置換するアプリである。

## 2. 基本コンセプト

青空文庫の短篇作品を短い断片に分割し、タイムライン形式で表示する。
各断片は投稿のように見えるが、SNS投稿ではなく作品本文である。

## 3. アカウント概念

1アカウント = 1作品。

著者単位ではなく作品単位で管理する。

例：

- 太宰治『女生徒』 = 1作品アカウント
- 夏目漱石『夢十夜』 = 1作品アカウント
- 芥川龍之介『蜜柑』 = 1作品アカウント

## 4. タイムライン

ホームタイムラインには、フォロー中の作品アカウントの断片が混在して表示される。

ただし、各作品の内部順序は崩さない。

## 5. タイムライン上の一投稿表示

タイムライン上では読むために必要な情報だけを表示する。

### 表示する

- 作品名
- 本文断片
- 必要に応じて著者名

### 表示しない

- アイコン
- 投稿日時
- アカウントID
- いいねアイコン
- しおりアイコン
- 引用アイコン
- 返信
- リポスト
- エンゲージメント数

### 操作

タイムライン上の断片カード全体をタップすると、原則として作品ページへ移動する。
いいね、しおり、引用は個別ページで行う。

例外：

- ホームタイムラインの `作品追加` カードは、作品ページの先頭を表示してよい
- ホームタイムラインの `最新しおり` と `いいね追加` の出来事カードは、作品ページ内の該当断片へ直接ジャンプしてよい
- 作品別タイムラインでは、右下の断片番号表示だけを状態トグルとして扱ってよい
- 状態は `なし → しおり → いいね → なし` の順で循環してよい
- `しおり` は現在位置、`いいね` は気になる断片の付箋として扱う

## 6. 一投稿あたりの本文仕様

- 標準：60〜160字
- 最大：220字
- 表示行数：2〜5行程度
- 長すぎる断片は作らない
- ルビや圏点の途中で分割しない

## 7. 断片個別ページ

個別ページでは、以下を表示する。

- 作品名
- 著者名
- 本文断片
- 前の断片
- 次の断片
- いいね
- しおり
- 引用として保存
- 作品ページへ

補足：

- しおりは作品ごとに常に1つだけ保持する
- 新しい断片でしおりを実行したら、同一作品内の古いしおりは新位置へ置き換わる
- いいねは読書中の付箋として扱い、作品内で複数保持してよい

## 8. 投稿機能

通常投稿は禁止。
ユーザーが保存できるのは引用のみ。

禁止：

- 日記
- 感想
- 愚痴
- 独り言
- 返信
- リポスト

## 9. 青空文庫取り込み

青空文庫のルビありZIPを直接取り込む。
ユーザーにZIPの手動解凍をさせない。

流れ：

1. ZIPを選択またはドラッグ＆ドロップ
2. ZIP内txtを抽出
3. Shift_JISとして読み込む
4. ヘッダ・フッタを除去
5. ルビ・圏点を解析
6. 本文を断片化
7. プレビュー
8. 作品アカウントとして保存

## 10. ZIP対応

外部ZIPライブラリは使わない。
青空文庫ZIPに必要な範囲だけ最小実装する。

対応：

- .zip
- .txt
- 圧縮方式 0: store
- 圧縮方式 8: deflate

非対応：

- パスワード付きZIP
- ZIP64
- 分割ZIP
- 巨大ZIP
- 複数作品入りZIP
- 画像入りZIP

## 11. 青空文庫記法

必須対応：

- ルビ
- 圏点

後回し：

- 字下げ
- 改ページ
- 外字注記
  ただし、`※［＃「...」、第3/4水準...］` や `※［＃二の字点、1-2-22］` のように
  JIS X 0213 の面区点番号を持つものは、対応する実字へ置換してよい
  `※［＃「...」、U+....、...］` 形式で Unicode 番号が明示されたものも実字へ置換してよい
  解決できない注記は原文を残す
- 見出し注記
  大見出し、中見出し、小見出しは表示に反映し、後の目次機能で使える構造情報として保持する
  字下げ注記は見出しの補助情報としてのみ扱い、大きな字下げ量をそのまま版面再現しない
- 傍線
- 罫囲み
- 図版
- 入力者注
- 校正者注

## 12. 保存

初期実装ではブラウザ内保存。

- IndexedDB: 作品、断片、いいね、しおり、引用
- localStorage: 軽い設定

軽い設定には、本文サイズのブラウザ別保存を含めてよい。

## 13. エクスポート/インポート

JSONエクスポートとインポートを必須にする。
端末移行とバックアップのため。

## 14. GitHub Pages

静的WebアプリとしてGitHub Pagesで運用する。
ビルド不要。
初期は `main` ブランチのルート `/` から公開する。
